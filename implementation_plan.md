# 分享体验与卡片重构计划 (Plan for Share Experience and Card Redesign)

针对您提出的两个问题，我制定了相关的调整方案。

## 1. 分享按钮位置调整方案

当前分享按钮（`> EXECUTE_SHARE_IMAGE`）放置在最底部的 `EXECUTE_EXTERNAL_UPLINK`（外部跳转）区域。为了提升点击率并符合用户直觉，我们可以将其上移。以下是几个方案供您选择：

**方案 A：海报右上角 (沉浸式 / 悬浮图标)**
- 将分享按钮从文本按钮修改为一个极简的“分享图标” (例如类似系统分享的箭头图标)。
- 放置在 `dossier-visual` (海报容器) 的右上角，采用绝对定位悬浮在海报上方，配合毛玻璃效果。
- **优点**：最醒目，不占用内容区空间，符合现代移动端阅读习惯。

**方案 B：标题/副标题右侧 (紧凑型)**
- 将分享按钮放置在 `dossier-title` (例如右侧或副标题同行)。
- **优点**：直接与剧集核心信息绑定，在用户刚看完标题和评分即可随时分享。

**方案 C：核心数据块下方**
- 放置在 `dossier-metrics` (评分和上映日期) 和 `> CORE_BRIEFING` (导演/主演) 的中间。作为一个突出的全宽或半宽胶囊按钮。
- **优点**：符合传统的向下阅读流，用户看完基本信息后可以顺手分享。

> **您更倾向于哪个按钮放置方案？(推荐方案A)**


## 2. 分享卡片样式重构与数据补充

当前分享图片存在几个问题：
1. 简介过长时超出卡片底部范围。
2. 缺失导演和主演信息。
3. 整体视觉只使用了黑蓝渐变色和发光线条，缺乏层次感和“高级感”。

为了解决这些问题，建议对 `createShareImageFile` (Canvas 绘制逻辑) 进行重构。以下是几种图片排版与视觉优化方案：

**方案 A：赛博朋克 / 数据终端风格 (Cyberpunk Data Terminal) - 推荐**
- **布局重构**：横向划分布局。左侧为完整的长海报（不裁剪裁断），右侧为信息区。
- **排版优化**：
  - 加大标题字体，加入导演/主演数据版块（限制各显示前3位以防超长）。
  - **简介截断**：通过计算 Canvas 绘制行数，对简介区域设置严格的高度限制。当文字超过最大允许行数时，末尾显示 `...`，绝对杜绝内容超出画面边界。
- **视觉**：背景采用网格阵列 (Grid) 与暗色 CRT 扫描线叠加，关键数据（评分等）使用荧光色框选（极简线条），突出“雷达扫描”和科幻终端的酷炫感。

**方案 B：极简胶片风 (Cinematic Minimalist)**
- **布局重构**：大留白居中布局。背景使用海报的**高斯模糊**作为底色，上方叠加一层深色半透明遮罩。
- **排版优化**：海报居中显示。下方紧接标题、评分、类型。导演和演员使用较小的副标题形式显示。
- **简介截断**：只保留最多 3 行的极简简介（超过自动 `...`），将更多空间留给海报和核心演职员数据。
- **视觉**：类似豆瓣分享或 Apple Music 分享卡片的高级毛玻璃质感，整体优雅精致，去“硬核代码”感。

**方案 C：票条风格 (Receipt / Ticket Style)**
- **视觉形式**：将分享卡片设计成一张电影票或收据的样式，带有虚线边缘或缺口。
- **排版**：顶部是雷达Logo和打孔设计；中间是小尺寸海报 + 关键数据（评分、导演、主演）；最下方是简介（像票据的备注一样严格限制行数），底部是生成日期和剧集条形码（可用随机竖线模拟）。
- **优点**：非常独特且有记忆点，适合社交媒体传播。
